2. Number数据的存储

JS底层采用浮点数存储Number类型数据，遵循IEEE 754标准。JS采用8字节(64位)存储浮点数，具体格式为：

二进制数据位置 存储内容
0-51 小数
52-62 指数
63 符号位

详细存储格式请参考How numbers are encoded in JavaScript

3. 整数相关的操作

在JS中，所有的整数操作（加、减、乘、除、乘方、开方、位运算）在实际执行时均遵循如下模式：

  • 将操作数转换为浮点数
  • 操作数转为整数
  • 计算
  • 将整数结果转换为一个浮点数

计算过程中会将浮点数转为整数，而这里的整数有4种类型：

类型 取值范围 应用场景
Integer ${[-2^{53}, 2^{53}]}$ 用于大部分的整数操作
Uint16 ${[0, 2^{16}-1]}$ 字符计算(character codes)
Uint32 ${[0, 2^{32}-1]}$ 数组长度
Int32 ${[-2^{31}, 2^{31}-1]}$ 位操作、无符号移位操作、按位取反

4. 解答文章开头的问题

第一个问题

第一步：由于是移位运算，1会转换为Int32类型（取值范围是${[-2^{31}, 2^{31}-1]}$）
第二步：计算1<<32。 1<<32 = 1<<(32%32) = 1<<0 = 1。（左移里一个比较特殊的情况是当左移的位数超过该数值类型的最大位数时，编译器/解释器会用左移的位数去模类型的最大位数，然后按余数进行移位。JS从现象看遵循该规则）
第三步：将计算结果转为浮点数
